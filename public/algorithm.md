# 美股流动性看板 — 总体逻辑与算法

> 线上:https://macro-liquidity-dashboard.pp-account.workers.dev
> 本文件描述当前实现(`src/metrics.ts` + `src/service.ts`)的完整逻辑与公式。

---

## 0. 一句话

**用美联储的"钱多钱少"判断美股顺不顺风。** 把 Fed 资产负债表、财政部账户、逆回购等宏观数据合成一个**净流动性**指标,再叠加利率/信用/美元/波动等因子,算出一个 **0–100 顺风指数**和一个 **偏多 / 偏空 / 中性** 的红绿灯结论。

---

## 1. 它回答的问题

不预测明天涨跌,而是回答:**"现在的宏观流动性环境,对美股是顺风还是逆风?"** —— 这是一个**慢变量、仓位级**的判断(周/月尺度),不是日内择时。

核心信念:**钱(流动性)往市场流 → 风险资产有底;钱被抽走 → 逆风。** 收益率"高"本身不杀股市,收益率"快速上冲"才杀。

---

## 2. 数据从哪来(口径隔离)

两套数据,**故意分开**,互不污染:

### 2.1 FRED = 历史与逻辑的唯一真相源(dated,入库)
所有打分逻辑只用 FRED 的"带日期、可回溯"数据。每日(现为每 3 小时)cron 增量拉取、存入 D1:

| 序列 | 含义 | 频率 | 单位处理 | 用途 |
|---|---|---|---|---|
| `WALCL` | Fed 总资产 | 周(三) | 百万 → ÷1000 = 十亿 | QE/QT 表头 + 净流动性 |
| `WTREGEN` | TGA 财政部账户 | 周(三) | 百万 → ÷1000 = 十亿(同 WALCL,H.4.1) | 净流动性减项 |
| `RRPONTSYD` | 隔夜逆回购 | 日 | 十亿(原样) | 净流动性减项 |
| `RPONTSYD` | 隔夜回购 | 日 | 十亿(原样) | 应急注水(尾部) |
| `SOFR` | 担保隔夜融资利率 | 日 | % | 资金压力 |
| `IORB` | 准备金利率 | 日 | % | 资金压力基准 |
| `BAMLH0A0HYM2` | 高收益债利差 HY OAS | 日 | % | 信用 |
| `DGS10` | 10 年美债收益率 | 日 | % | 利率冲量(用变化率) |
| `VIXCLS` | 波动率指数 VIX | 日 | 指数 | 波动 |
| `DTWEXBGS` | 美元广义指数 | 日 | 指数 | 美元 |
| `SP500` | 标普 500 | 日(滞后1天) | 指数 | 历史叠加图的 SPX 线 |

> ⚠️ 单位坑:`WALCL`/`WTREGEN` 来自 H.4.1,是**百万**;`RRPONTSYD`/`RPONTSYD` 来自临时公开市场操作,是**十亿**。混了会让净流动性算成负的。

### 2.2 实时价格 = 只贴顶部"此刻"数字(不入库)
SPX / VIX / DXY / 10Y 走 **Yahoo(主)/ Stooq(备)**,每次开页面现拉、缓存 ~60s。**只用于顶部显示,绝不进打分** —— 这样宏观慢逻辑不会被日内价格噪音晃动。

## 3. 总体逻辑:三层判断

```
            ┌─────────────────────────────────────────────┐
   FRED ──▶ │  ① QE/QT 表头   ② 净流动性(核心)  ③ 七因子   │
            │      (regime)        (方向/趋势)     (0-100)  │
            └───────────────┬─────────────────────────────┘
                            ▼
              0-100 顺风指数  ─分带+迟滞─▶  偏多/偏空/中性 红绿灯 + 理由
```

### ① QE/QT 表头（regime）
看 Fed 资产负债表(WALCL)**13 周(一个季度)的变化**:扩表(QE)/ 缩表(QT)/ 中性。这是"水龙头开还是关"的大背景。

### ② 净流动性（核心驱动）
> **净流动性 = WALCL − TGA − RRP**

直觉:Fed 资产负债表是放进系统的总钱;但 TGA(财政部存在 Fed 的钱)和 RRP(货基停在 Fed 的钱)是**被锁在系统外、没进市场**的部分。减掉它们,才是真正"在市场里流动"的钱。

**最关键的洞察 —— 背离:** Fed 可以一边缩表(QT),一边因为 RRP/TGA 大幅下降而让净流动性**仍在上升** → 这正是 2022–2024"缩表却大涨"的真相。本看板把这个背离**显式讲出来**(见 §4.7),不让"QT"这个标签误导你。

### ③ 0-100 顺风指数
把净流动性趋势 + 7 个因子各自打成 0–100 的子分,加权求和。越高越顺风。

## 4. 算法细节(逐个公式)

### 4.0 通用工具
```
clamp(x)        = max(0, min(100, x))                  # 钳到 [0,100]
linMap(x, a, b) = clamp( (x - a) / (b - a) * 100 )     # 把 [a,b] 线性映射到 [0,100]
sma(v, n)       = 最近 n 个值的均值(不足 n 个返回 null)
asOf(series, d) = series 中日期 ≤ d 的最后一个值(前向填充;升序)
```
> `linMap` 里 a、b 哪个对应"好"很关键:`linMap(x, 30, 12)` 表示 x=30→0 分、x=12→100 分(越小越好)。

### 4.1 净流动性序列与趋势(真·周度)
- **周度净流动性序列**:遍历 WALCL 的每个周三日期 `d`,算 `WALCL(d) − asOf(TGA,d) − asOf(RRP,d)`。
  - 用周度序列算趋势,避免了"在日线图上 20 周其实算成 4 周"的周期漂移 bug。
- **趋势线** `netliq_trend = sma(周度净流动性, 13)`(13 周 ≈ 一季度)。
- **方向** `netliqDirection`:
  ```
  rel = (最新净流动性 − 13周SMA) / max(1, |13周SMA|)
  rel >  0.002 → UP(在升)
  rel < -0.002 → DOWN(在收)
  否则         → FLAT(走平)        # 0.2% 死区,防抖
  ```

### 4.2 QE/QT 表头
```
d13 = WALCL_weekly[最新] − WALCL_weekly[13周前]   # 不足14个点 → NEUTRAL
d13 >  +50($B) → QE(扩表)
d13 <  −50($B) → QT(缩表)
否则            → NEUTRAL(横住)     # ε=50亿美元 死区(初值,待校准)
```

### 4.3 七个因子打分(每个输出 0–100,越高 = 对股市越顺风)

| 因子 | 权重 | 公式 | 直觉 |
|---|---|---|---|
| **净流动性趋势** `netliqTrend` | **0.35** | `0.5·aboveMA + 0.5·linMap(4周斜率, −200, +200)`,其中 `aboveMA = 60 if 最新>13周SMA else 40` | 钱在流入且在均线上方 = 顺风(最大权重) |
| **QE/QT 表头** `qeqt` | 0.15 | `QE→80, NEUTRAL→55, QT→30` | 水龙头方向 |
| **信用** `credit` | 0.15 | `(1 − 历史分位(HY OAS)) · 100` | 高收益债利差越低(越平静)= 越顺风 |
| **资金面** `funding` | 0.10 | `linMap(SOFR−IORB, 0.10, 0.0)` | SOFR-IORB ≤0 平静→100;冲到 +10bp→0(钱荒) |
| **利率冲量** `rates` | 0.10 | `linMap(Δ10Y(20交易日), +0.5, −0.5)`;无数据→50 | 收益率**快速上冲**=逆风(用变化率,不是绝对水平) |
| **美元** `dollar` | 0.08 | `linMap(z, +1.0, −1.0)`,`z=(DXY−SMA200)/σ200`;不足200点→50 | 美元相对200日均值猛涨(z 高)=逆风 |
| **波动** `vol` | 0.07 | `linMap(VIX, 30, 12)`;无数据→50 | VIX 低=顺风(12→100, 30→0) |

> "历史分位"= 该值在**全部历史**里小于等于它的占比(`percentileRank`)。
> 缺数据时:`credit`/`funding` 缺→子分给 50(中性);`rates`/`vol`/`dollar` 缺→各自函数返回 50。

### 4.4 加权 → 0-100 顺风指数
```
score = clamp( 0.35·netliqTrend + 0.15·qeqt + 0.15·credit
             + 0.10·funding + 0.10·rates + 0.08·dollar + 0.07·vol )
```
(权重和为 1。)

### 4.5 红绿灯结论 + 迟滞
```
score > 55 → 偏多 (BULLISH)
score < 45 → 偏空 (BEARISH)
45 ≤ score ≤ 55 → 维持上一日结论(没有上一日则 中性)   # 10分死区 = 迟滞,防止边界反复翻
```
回填/刷新时,`prev`(昨日结论)按时间顺序往后传,所以死区里的"维持"是真的跨天持续,不是单次。

### 4.6 P0–P3 展示分组
0-100 是连续分;P0–P3 只是**因子明细表的展示分组**(沿用原 Pine 习惯),每组给个通过/不通过:
```
P0 风控   = rates≥50 且 funding≥50 且 credit≥50
P1 流动性 = netliqTrend≥50 或 qeqt≥50
P2 美元   = dollar≥50
P3 结构   = vol≥50
```

### 4.7 理由文案(含背离)
```
"Fed {扩表/缩表/横住}、净流动性{在升/在收/走平} → 环境{偏多/偏空/中性}{背离提示}"
背离提示: QT 且方向UP → "(缩表却放水,留意背离)"
          QE 且方向DOWN → "(扩表却收水,留意背离)"
```

## 5. 一个真实算例(2026-06-17 线上快照)

原始数据:`WALCL=6736.4B`, `TGA=880.7B`, `RRP=6.8B` → **净流动性 = 6736.4 − 880.7 − 6.8 = 5848.9B(≈$5.85T)**
`HY OAS=2.63%`, `SOFR−IORB=−0.02`, `Δ10Y`(10Y=4.49%), `DXY=119.5`, `VIX=18.4`

各因子子分 → 加权:
```
netliqTrend 47.7 × 0.35 = 16.70
qeqt        80.0 × 0.15 = 12.00
credit      98.2 × 0.15 = 14.73   (HY 利差处历史极低分位 → 信用很平静)
funding    100.0 × 0.10 = 10.00   (SOFR-IORB 为负 → 资金极宽松)
rates       46.0 × 0.10 =  4.60
dollar      59.3 × 0.08 =  4.74
vol         64.2 × 0.07 =  4.49
                       ───────
                 score = 67.26  →  > 55  →  偏多 (BULLISH)
```
regime = **QE**(WALCL 13周净增 > 50亿),方向 = FLAT(净流动性走平)。
结论文案:**"Fed 扩表、净流动性走平 → 环境偏多"**。

## 6. 数据流水线

```
cron(每3小时)
  └─ runIngest(rebuildAll=false)
       ├─ 对每个 FRED 序列:从 D1 已有最大日期增量拉取 → upsert 到 observations
       └─ 重算最近 14 天的 daily_snapshot(用 computeSnapshot)→ upsert

一次性全量回填 runIngest(rebuildAll=true):
       └─ 按 WALCL 周线日期采样重算全历史快照(避免逐日 O(N²) 爆 CPU)

前端(每次开页面)
  ├─ GET /api/snapshot  → D1 最新快照 + 实时价格(合并)
  └─ GET /api/history   → D1 历史序列(净流动性 / SPX 叠加图)
```
- 存储:D1 两张表 `observations`(FRED 原始长表)/ `daily_snapshot`(每日算好的状态)。
- `computeSnapshot` 是**纯函数**,21+ 个单测覆盖(`src/metrics.ts`)。

## 7. 设计取舍 · 已知局限 · 校准项

- **实时度上限 = 每日**:FRED 数据本身日频+滞后,资产负债表更是周频。顶部价格已实时,但宏观判断不会日内跳动(有意为之)。
- **历史窗口**:当前从 `2020-01-01` 回填(覆盖 COVID 放水→2022 缩表→背离),够展示也够算趋势;窗口越长回填越慢。
- **阈值是初值,待校准**:7 个因子的权重、`QEQT_EPSILON_B=50`、各 `linMap` 边界、verdict 分带(55/45),都是**先验初值**,计划上线后用历史回测(regime 对未来 SPX 收益的命中率)来校准。
- **广度缺失**:涨跌家数(ADD)无干净免费源,MVP 未纳入(P3 只用 VIX)。
- **美元口径**:实时顶部用真 ICE DXY(`DX-Y.NYB`),因子/历史用 FRED 广义美元(`DTWEXBGS`)——走势一致、绝对值不同,有意取舍(dated 历史优先)。
