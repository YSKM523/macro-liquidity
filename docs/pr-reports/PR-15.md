# PR-15 — Purged validation and outcome taxonomy

## Status

Implementation and local verification are complete; independent rereview is pending. The change is additive and does not alter the Champion model or stored snapshots.

## Implemented

- `PURGED_VALIDATION_V1` uses strictly increasing unique ISO dates, 13-week interval labels, outcome-overlap purging, and a 91-calendar-day embargo. Irregular spacing is accepted only when the first eligible outcome is no more than 14 days late.
- Every published test label includes `signalDate` and `outcomeDate`; folds report train/test label bounds, purge/embargo counts, overlapping and greedily interval-non-overlapping counts.
- Champion metrics use the persisted score, persisted formal verdict, and the existing `DASHBOARD_EXPOSURE_TIERS_V1` target derived from the immutable snapshot VIX proxy. No verdict is reconstructed from an isolated score.
- Typed metrics cover direction, formal verdict, risk precision/downside recall, Spearman IC, and tail recall/precision. Minimum samples, zero variance, missing formal fields, pending maturity, and partial legacy calibration are null with explicit status.
- Fold q10 and diagnostic IC weights consume only matured, post-purge training outcomes. Diagnostic fitted direction/IC is separate and is not represented as Champion performance. Aggregate tail counts combine each fold's own training-only q10 confusion matrix; no final-fold threshold is applied retrospectively.
- The forward holdout is registered at `2026-07-23` with a deterministic protocol digest and pre-holdout outcome cutoff `2026-04-23`. Its frozen training identity cannot move when post-registration rows append. It remains `PENDING_MATURITY` until five 13-week labels mature.
- Governed and migration-0010 legacy PIT history can be evaluated together under explicit counts and `PARTIAL_LEGACY`. Legacy q10 calibration is disabled as `PARTIAL_LEGACY_CALIBRATION`; malformed provenance/non-PIT rows fail closed, and every post-registration holdout signal must be governed.
- `/api/walkforward`, `/api/robustness`, `/api/v1/robustness`, and the robustness dashboard expose the additive object while preserving all prior fields and values.

## Known limitations

- The genuine holdout begins tomorrow relative to registration and has no mature 13-week outcomes. No performance claim is made for it.
- Existing migration-0010 snapshots lack governed model identity. Retrospective non-tail metrics remain useful only with `PARTIAL_LEGACY` disclosure; tail calibration is intentionally null until governed training evidence exists.
- Weekly outcome matching permits up to 14 days of irregular lateness, matching the existing historical forward-return availability rule.
- This PR does not implement BT-08 score monotonicity or extreme-event stress testing.
