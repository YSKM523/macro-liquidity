## 3. Design decisions

- Canonical JSON is key-sorted and hashed with deterministic SHA-256. The frozen golden digest is `17ad1ca8854b0fbd8e56d6255b7ee2f4fe8a85ae1a95a328ade46ffdff02a0cf`; it matches Node crypto and changes under configuration drift. Event-backtest execution, benchmark, accounting, annualization, conversion, compatibility, and reporting conventions are all explicit inputs to that identity. Invalid/missing deployment SHA is `LOCAL_UNCONFIGURED`, never a fake commit.
- Migration 0010 is additive; existing rows become `LEGACY_UNVERSIONED`. v1 history returns those rows with `LEGACY` status and null unknown provenance alongside governed rows; it never invents identity.
- Backtest/robustness v1 responses distinguish runtime model identity from the validated persisted identities of returned snapshot rows.
- Every admin attempt atomically reserves a hashed source bucket before authentication/audit work. Admin auth accepts exact legacy bearer or configured Access service-token credentials. Full rebuild needs the independent exact `FULL_REBUILD` header.
- Typed provider failures pass through the cache loader so they increment the circuit. Stale or failed stress is always exposed as `UNKNOWN`, never stale `NORMAL` guidance.
- Backup has no side effect without `--execute`; production adds exact confirmation. D1 remains remote-only in execute mode while R2 uses its valid object command. Restore applies all migrations, generates a full schema/data equivalent export, and restores it into a second ephemeral D1 without SQL whitespace folding.

## 6. Known limitations

- Staging D1 is deliberately `REPLACE_WITH_STAGING_D1`; staging and production deployment remain unverified.
- Structured logs and alert delivery are locally tested, but remote log search and real recipient delivery require authorized configuration/deployment.
- The restore drill uses representative generated data and a real equivalent schema/data export; it is not evidence that an actual production backup has been restored.
- Cache is per Worker isolate, not a distributed quote cache. Stale stress fails closed.
- Historical rows cannot acquire provenance retroactively. V1 history labels them `LEGACY` and nulls unknown identity/provenance, so governed-only comparisons must filter by status.
- The Access hostname/path/method policy and environment approvals are documented but cannot be proven without authorized Cloudflare/GitHub configuration inspection.
